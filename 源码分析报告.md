## 二、详细技术分析

### 1. IMGui菜单自定义控件案例

#### 核心文件
- `ImGuiLoad.m`: 负责显示ImGui界面
- `ImGuiDrawView.mm`: ImGui视图控制器，使用Metal渲染

#### 实现原理
```objective-c
// ImGuiLoad.m 关键代码
- (void)show {
    if (!_vna) {
        ImGuiDrawView *vc = [[ImGuiDrawView alloc] init];
        _vna = vc;
    }
    [ImGuiDrawView showChange:true];
    // 直接添加到应用主窗口
    [[UIApplication sharedApplication].windows[0].rootViewController.view addSubview:_vna.view];
}
```

#### 优点
✅ 代码简单，易于理解  
✅ 不依赖私有API  
✅ 适合应用内悬浮显示  
✅ 触摸交互正常

#### 缺点
❌ 不是真正的系统级悬浮窗口  
❌ 依赖应用的主窗口，应用退出时窗口消失  
❌ 可能被其他视图遮挡  
❌ 无法跨应用显示

### 2. SysView巨魔安装后悬浮例子

#### 核心文件
- `AppDelegate.m`: 创建悬浮Scene和窗口
- `Window.m`: 自定义窗口类，实现触摸穿透
- `UIKitPrivate.h`: 私有API声明

#### 实现原理
```objective-c
// AppDelegate.m 关键代码
- (UIRootWindowScenePresentationBinder *)createFloatingSceneWithIdentifier:(NSString *)sceneIdentifier {
    // 1. 创建Scene定义
    FBSMutableSceneDefinition *definition = [FBSMutableSceneDefinition definition];
    definition.identity = [FBSSceneIdentity identityForIdentifier:sceneIdentifier];
    
    // 2. 设置Scene参数
    UIMutableApplicationSceneSettings *settings = [UIMutableApplicationSceneSettings new];
    settings.level = 10000; // 高层级
    settings.foreground = YES;
    settings.canShowAlerts = YES; // 支持键盘
    
    // 3. 创建Scene和绑定器
    FBScene *scene = [sharedInstance createSceneWithDefinition:definition initialParameters:parameters];
    UIRootWindowScenePresentationBinder *sceneBinder = [[UIRootWindowScenePresentationBinder alloc] initWithPriority:1000 displayConfiguration:settings.displayConfiguration];
    [sceneBinder addScene:scene];
    
    return sceneBinder;
}

// 创建两个窗口：响应触摸和穿透触摸
self.overlayWindowIsTouch = [[UIWindow alloc] initWithFrame:UIScreen.mainScreen.bounds];
self.overlayWindowIsTouch.windowLevel = UIWindowLevelStatusBar + 1000;
self.overlayWindowIsTouch.userInteractionEnabled = YES;
[self.overlayWindowIsTouch makeKeyAndVisible];

self.overlayWindowNoTouch = [[Window alloc] initWithFrame:UIScreen.mainScreen.bounds];
self.overlayWindowNoTouch.windowLevel = UIWindowLevelStatusBar + 999;
self.overlayWindowNoTouch.userInteractionEnabled = NO;
```

#### 优点
✅ 真正的系统级悬浮窗口  
✅ 可以跨应用显示  
✅ 窗口层级高，不易被遮挡  
✅ 支持键盘和弹窗  
✅ 支持触摸穿透（通过`_ignoresHitTest`）

#### 缺点
❌ 依赖大量私有API，可能随iOS更新失效  
❌ 需要特殊权限（如TrollStore）  
❌ 代码复杂度高  
❌ 可能被App Store拒绝（如果用于上架）
